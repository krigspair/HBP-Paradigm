## 1. Статус П1 (Энактивизм / Прагматический процессуализм)

### 1.1 Вопрос
Является ли П1 («реальность — процесс, а не субстанция»)
фальсифицируемой гипотезой или метатеоретическим допущением?

### 1.2 Ответ
П1 — метатеоретическое допущение (Lakatosian hard core),
не фальсифицируемое напрямую.

Обоснование:
(a) П1 задаёт онтологическую рамку, в которой формулируются
    все остальные постулаты HBP. Фальсификация требует
    альтернативной онтологии (субстанциализм), но такая
    замена меняет всю программу, а не отдельный постулат.

(b) Тем не менее П1 операционально тестируем через
    сравнительные предсказания: динамические переменные
    (аттракторы, фазовые переходы, switching rate)
    должны предсказывать исходы лучше, чем статические
    черты (trait anxiety, big-5 neuroticism).
    → Это формулируется как KC-0 операциональный тест
      (не kill criterion, но boundary condition).

### 1.3 Следствия для System Map
- П1 остаётся в hard core (Tier 0) без изменений.
- В System Map v1.4 добавить примечание:
  «П1 — метатеоретическое допущение; операциональный
   тест: динамические предикторы > статических черт
   (ΔR² критерий, таблица в P1.2)».

## 2. Статус П12 (Прагматический процессуализм)

### 2.1 Вопрос
П12 утверждает, что «истинность теоретического конструкта
определяется его прагматической полезностью в клиническом
процессе». Как соотносится это с научным реализмом
(критерием фальсифицируемости)?

### 2.2 Анализ
П12 задаёт прагматический критерий оценки конструктов,
не заменяя фальсификацию, а дополняя её:

(a) Уровень 1 (Фальсификация, Popper):
    Конструкт исключается из модели, если он не добавляет APV
    (ΔR² < порога, Kill Criteria активируются).
    → Это уровень эмпирической элиминации.

(b) Уровень 2 (Прагматическая оценка, П12):
    Конструкт сохраняется в клинической практике даже
    при низком APV, если он функционален как
    клиническая метафора (пример: «аттрактор» как
    объяснительный инструмент для клиента).
    → Это уровень практической полезности.

(c) Разрешение противоречия:
    HBP явно разделяет два уровня:
    — Научная модель: управляется APV + Kill Criteria
    — Клиническая практика: управляется П12
    Нет конфликта: П12 не отменяет фальсификацию,
    а устанавливает критерий для клинического использования
    конструктов, прошедших или не прошедших APV-тест.

### 2.3 Следствия для System Map
- П12 сформулировать явно как «прагматический фильтр
  второго уровня» (после APV).
- Добавить в System Map v1.4, раздел I, примечание:
  «П12 применяется после APV-скрининга:
   APV-pass → конструкт в научной модели И в практике;
   APV-fail → конструкт только в клинической практике
   (если клинически функционален) с пометкой
   «heuristic, not validated»».

## 3. Таблица: Типология конструктов HBP по П12

| Тип | APV | П12-полезность | Статус в HBP |
|-----|-----|----------------|--------------|
| A   | +   | +              | Tier 1/2 (научная модель + практика) |
| B   | +   | −              | Tier 1/2 (научная модель, но не практика) |
| C   | −   | +              | Tier 3 или heuristic |
| D   | −   | −              | Элиминация |

Примеры:
  Тип A: TTOM (T1-4) — тестируется + клинически операционален
  Тип C: IIT / Phi — низкий APV, но метафора «интеграции» полезна
  Тип D: Telomere → исход (слабые данные + неоперационален)

========================================================================
ВЫВОД
========================================================================

1. П1 — hard core допущение, операционально тестируемое
   через KC-0 (dynamic > static predictors).
2. П12 — прагматический фильтр второго уровня,
   не противоречащий фальсификации, а дополняющий её.
3. Система двух уровней (APV → П12) делает HBP
   одновременно фальсифицируемой и клинически гибкой.
